# Funding Rates Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add opt-in perpetual **funding** — a discrete, signed cash flow per position every funding interval — to the paper-trading engine, building on the live leverage/liquidation core.

**Architecture:** Two pure broker functions (`funding_payment`, `funding_due`) plus loop wiring in bot + backtest. Funding is an **account-level cash flow** (debits/credits cash, clamped ≥ 0), applied once per `funding_interval_hours` crossing, gated on `funding_rate != 0`. The entry-based liquidation price is unchanged. `funding_rate = 0` (default) ⇒ byte-identical to today.

**Tech Stack:** Python 3.14 engine (pytest, pydantic, pyyaml).

## Global Constraints

- **TDD always:** red → green per step. Run the named command, confirm the stated result before moving on.
- **`funding_rate = 0` (default) must stay byte-identical** to current live behavior: no time tracking, no cash flow, no `last_funding_ts` written — the whole path is gated off.
- **Sign convention (verbatim):** `funding_payment = −funding_rate · qty · price`. Positive rate ⇒ longs pay (negative delta), shorts receive (positive delta). Flat ⇒ 0.
- **Funding cash flow is clamped `≥ 0`** (`max(0.0, …)`), mirroring the apply_fill bad-debt guard. It must never crash or drive cash negative.
- **The risk gate, `apply_fill`, and the entry-based liquidation price are NOT changed.** Funding is layered on as a cash flow only.
- **No new dependencies.** No change to `models`/`indicators`/`strategies`/`sentiment`/`metrics`/`datafeed`/`market`/`llm`.

def funding_payment(position: Position, price: float, funding_rate: float) -> float:
    """Signed cash delta from one funding charge: -rate*qty*price.

    Positive rate -> longs pay (negative delta), shorts receive (positive). Flat -> 0.
    """
    return -funding_rate * position.qty * price


def funding_due(last_ms, now_ms, interval_hours: float) -> bool:
    """True once a funding interval has elapsed since last_ms (epoch ms).

    last_ms is None -> False (first observation initializes the clock, never pays).
    """
    if last_ms is None:
        return False
    return (now_ms - last_ms) >= interval_hours * 3_600_000
```

## Funding

Set `risk.funding_rate` (per `risk.funding_interval_hours`, default 8h) to charge perpetual
**funding** on open positions: a positive rate means **longs pay shorts** (and vice versa),
`funding_rate · notional` each interval. It debits/credits cash (clamped ≥ 0) and so erodes or
boosts equity for as long as a position is held — the holding cost the leverage + liquidation model
was missing. `funding_rate = 0` (the default) is off and changes nothing. Funding is an account
cash flow here; it doesn't shift a position's isolated liquidation price (the realistic refinement,
and a live ccxt funding feed, are the upgrade paths).
```
